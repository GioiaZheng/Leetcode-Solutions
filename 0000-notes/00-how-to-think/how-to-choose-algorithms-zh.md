# 如何选择算法（中文版）

目标：看到新题后，能在 1~2 分钟内缩小到 1~3 个可行方向。

## 一、先看输入规模

- `n <= 20`：回溯/状态压缩常见。
- `n <= 2000`：O(n^2) 可能可过。
- `n <= 1e5`：优先 O(n)、O(n log n)。
- `n` 更大：考虑数学规律、位运算、线性构造。

## 二、根据题目信号选套路

- **连续区间**：滑动窗口、前缀和。
- **有序数组查找**：二分。
- **第 k 大/最小若干**：堆或快速选择。
- **图最短路**：Dijkstra / BFS（无权图）。
- **依赖关系**：拓扑排序。
- **全局最优且局部可证明**：贪心。
- **最优子结构**：动态规划。

## 三、先暴力再优化

先说出最直接解法，再问：
1. 是否重复计算？→ 记忆化/DP。
2. 是否可维护窗口状态？→ 滑窗。
3. 是否能把“查找答案”变成“判定答案”？→ 二分答案。

## 四、例子

**例 1：最长不重复子串**
- 信号：连续子串 + 去重。
- 方案：滑动窗口 + 哈希计数。
- 复杂度：O(n)。

**例 2：最小化最大值**
- 信号：答案具有单调性。
- 方案：二分答案 + `check(mid)`。

## 五、选择时的常见误区

- 误区 1：看到数组就上双指针（但题目可能要求全局最优）。
- 误区 2：看到“最大最小”就上 DP（很多其实是贪心/二分）。
- 误区 3：不先估算复杂度。

## 六、实战流程（推荐）

1. 划关键词（连续/最短/依赖/路径）。
2. 估复杂度上限。
3. 列 2 个候选算法并比较。
4. 先写最稳的版本。
5. 用边界样例验证。
