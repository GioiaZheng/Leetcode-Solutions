# 如何阅读 LeetCode 题目（中文版）

本笔记面向初学者，帮助你把“读懂题意”变成可执行的步骤。英文详版见：[how-to-read-problems.md](how-to-read-problems.md)。

---

## 核心目标

读题不是翻译故事，而是回答三个隐藏问题：

1. **输入结构**是什么？（数组/字符串/矩阵/链表/树/图）
2. **输出形式**是什么？（值/位置/序列/是否存在）
3. **输入与输出关系**是什么？（要找、要数、要验证）

只有这三点清晰了，才开始选算法。

---

## 步骤清单

### 1）先识别输入类型（最重要）
- 输入类型直接限制可用算法。
- 例如：
  - 字符串 → 滑动窗口 / 双指针 / 计数
  - 有序数组 → 二分查找
  - 树/图 → DFS/BFS/递归

### 2）抓住关键词
- contiguous / substring / subarray → 滑动窗口
- sorted → 二分
- at most / at least → 前缀和/滑窗
- maximum / minimum → 贪心 / DP
- number of ways → DP/组合

### 3）确定问题类型
- **找最值**（最大长度/最小成本）
- **计数**（方案数/子数组数）
- **验证**（是否存在/是否可行）

### 4）先别看约束
先理解关系，再看规模。规模只是筛选复杂度，不是解题思路。

---

## 小结

- 读题是技能，关键是“结构化提问”。
- 先输入结构，再关键词，再目标类型。
- 只有题意清晰，算法选择才不盲目。
