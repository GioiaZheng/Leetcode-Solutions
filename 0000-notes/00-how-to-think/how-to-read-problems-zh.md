# 如何读懂算法题（中文版）

很多“不会做”并不是不会算法，而是没有把题读透。以下是高效读题模板。

## 1. 用一句话重写题意

模板：
> 给定 ___，在 ___ 约束下，求 ___。

例：
> 给定整数数组，求和为 k 的连续子数组个数。

## 2. 标记关键词

重点词：
- 连续 / 不连续
- 至多 / 至少 / 恰好
- 任意顺序 / 保持原顺序
- 最长 / 最短 / 个数 / 是否存在

这些词直接决定算法方向。

## 3. 明确输入输出与限制

- 数据范围（n、多组数据）
- 数值范围（是否会溢出）
- 输出格式（一个值、多个下标、方案数量）

## 4. 先做样例“手推”

建议至少推 2 个：
- 官方样例
- 你自造边界样例（空、全重复、全负数等）

## 5. 抽象成标准模型

把题目语言翻译成算法语言：
- “每次只能向右扩展” → 双指针/滑窗
- “若 a 可做完才能做 b” → 有向图 + 拓扑排序
- “求最少步数” → BFS（无权图）

## 6. 例子：读题到建模

题：最短子数组和至少为 S。

- 关键词：最短、连续、至少。
- 建模：连续区间 + 单调性。
- 方案：前缀和 + 单调队列（或正数条件下滑窗）。

## 7. 读题检查清单

- [ ] 我能口头复述题目吗？
- [ ] 我知道输出的是值还是位置吗？
- [ ] 我考虑了不存在解的情况吗？
- [ ] 我的算法与“连续/顺序”要求一致吗？
