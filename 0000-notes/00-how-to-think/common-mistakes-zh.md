# 刷题常见错误（中文版）

这份笔记总结了新手到进阶阶段最容易犯的错误，并给出“纠错动作”。

## 1) 读题不精确

**表现**：把「子数组」写成「子序列」，把「恰好」写成「至少」。

**纠错动作**：
- 给题目关键词画线：连续/不连续、最小/最大、是否允许重排。
- 在纸上写 1 句自己的题意复述。

**例子**：
- 题目要“最长**连续**子数组”，你若用了排序就已经偏题。

## 2) 先写代码，后想状态

**表现**：边写边改，逻辑反复推倒。

**纠错动作**：先写三件事：
1. 输入规模与复杂度目标。
2. 需要维护的状态（如窗口计数、前缀和、最值）。
3. 状态转移（加入元素/删除元素后如何更新）。

## 3) 边界条件漏掉

重点检查：
- 空数组、长度为 1。
- 全相等、全递增/递减。
- 结果不存在（返回 -1、0、空列表）。
- 整数溢出（尤其 Java/C++）。

## 4) 复杂度不匹配

**表现**：n=1e5 还在用 O(n^2)。

**纠错动作**：
- 先根据 n 反推允许复杂度：
  - 1e5 通常 O(n) 或 O(n log n)
  - 2e3 左右可考虑 O(n^2)
- 再选算法：双指针/滑窗/哈希/前缀和/二分/堆。

## 5) 调试方式低效

**建议流程**：
1. 先用最小样例手推。
2. 打印关键状态（指针、计数器、dp 数组）。
3. 若答案不对，先定位“第一步出错的位置”。

## 6) 不会复盘

每题做完至少写三行：
- 这题核心模式是什么？
- 我第一次错在哪里？
- 下次看到什么信号应立即想到该模式？

## 快速自检清单

- [ ] 我有明确状态定义吗？
- [ ] 我能解释每一行更新语句的意义吗？
- [ ] 我验证了 3 个边界样例吗？
- [ ] 复杂度满足题目规模吗？
