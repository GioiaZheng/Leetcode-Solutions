# 哈希基础（中文版）

哈希的本质：用空间换时间，把“查找某状态是否出现过”降为近似 O(1)。

## 什么时候用哈希

- 需要快速判断“是否存在”。
- 统计频次。
- 记录某个前缀状态第一次出现的位置。

## 常用结构

- `set`：判重、存在性。
- `dict/map`：键值映射、计数。

## 例子：Two Sum

```python
seen = {}
for i, x in enumerate(nums):
    y = target - x
    if y in seen:
        return [seen[y], i]
    seen[x] = i
```

## 复杂度

- 平均：插入/查询 O(1)
- 最坏：可能退化，但工程上通常可接受。

## 易错点

- 更新时机：先查再放，避免同一元素重复使用。
- 键设计不当（如浮点误差）。
- 忘记冲突时需要“列表/计数”而非单值。
